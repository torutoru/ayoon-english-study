# 개발 기획서 — 아윤이 영어 스피킹 헬퍼

> 본 문서는 [spec.md](spec.md)의 요구사항을 실제 구현으로 옮기기 위한 개발 기획서다. 화면 디자인은 [test.html](test.html)을 기준으로 한다.

## 1. 기술 스택

| 구분 | 선택 | 비고 |
|------|------|------|
| 빌드/번들러 | Vite | 빠른 HMR, WASM 처리 용이 |
| 프레임워크 | React 19 + TypeScript | 화면 전환/상태 관리 |
| 스타일 | 단일 CSS (test.html 스타일 이식) | 별도 UI 라이브러리 없음 |
| AI | OpenAI `gpt-4.1-mini` | 브라우저 직접 호출 (`dangerouslyAllowBrowser`) |
| TTS | Web `SpeechSynthesis` | 영어 음성 |
| STT (기본) | Web `SpeechRecognition` | `en-US` |
| STT (폴백) | OpenAI Whisper (WASM) | 인식률 저하 시. **2차 마일스톤** |
| 저장소 | IndexedDB | 날짜별 대화 기록 |

> **참고**: Whisper WASM은 번들 크기·복잡도가 크므로 1차 구현에서는 `SpeechRecognition`만 적용하고, 폴백 인터페이스(`SttEngine`)만 미리 추상화해 둔다. Whisper 통합은 2차 마일스톤으로 분리한다.

## 3. 데이터 모델 (`types.ts`)

```ts
type Sender = '티쳐' | '학생';

interface Message {
  sender: Sender;
  msg: string;
}

interface DailyRecord {
  topic: string;
  messages: Message[];
  summary: string | null;   // 100자 이내, 이후 날짜 접속 시 lazy 생성
}
// IndexedDB: key = 'yyyy-mm-dd' (string), value = DailyRecord
```

## 4. IndexedDB 설계 (`services/db.ts`)

- DB명 `ayoon-english`, store `records` (keyPath 없음, key를 날짜 문자열로 직접 지정).
- API:
  - `getRecord(date): Promise<DailyRecord | undefined>`
  - `putRecord(date, record): Promise<void>`
  - `getAllRecords(): Promise<Array<{ date: string; record: DailyRecord }>>`
  - `getRecordsBefore(date): Promise<...>` — 오늘 이전 기록만 (요약 대상 조회)
- 오늘 날짜는 `config`의 헬퍼로 `yyyy-mm-dd` 포맷 통일.

## 5. OpenAI 서비스 (`services/openai.ts`)

브라우저에서 직접 호출. 키는 `import.meta.env.VITE_OPENAI_API_KEY`로 주입(`.env.local`). 모든 호출에 `model: 'gpt-4.1-mini'` 고정. 4개 함수:

1. `generateTopic(): Promise<string>` — 6살 아이용 랜덤 영어 회화 주제.
2. `generateQuestion(topic, history): Promise<string>` — 티쳐의 다음 영어 질문. 대화 history를 컨텍스트로 전달.
3. `evaluateAnswer(topic, history, question, answer): Promise<EvalResult>`
   - 반환: `{ correct: boolean; reply: string }`
   - `correct=false` → `reply`는 **한국어 교정**. `correct=true` → `reply`는 **다음 영어 질문**.
   - 구현은 JSON 응답(`response_format: json_object`)으로 분기 신뢰성 확보.
4. `summarize(messages): Promise<string>` — 100자 이내 한국어 요약.

프롬프트는 6살 아이 눈높이(쉬운 단어·짧은 문장·격려 톤)를 시스템 메시지로 명시.

## 6. 대화 루프 상태 머신 (`hooks/useConversation.ts`)

spec.md 1~8단계를 구현. 상태:

```
idle → topicLoading → speaking(TTS) → listening(STT) → evaluating(AI) → speaking ...
```

- 진입 시: `getRecord(today)` 존재하면 이어가기, 없으면 `generateTopic` → 첫 질문 생성 → 저장.
- 마이크 버튼: `listening` 시작 → STT 결과를 `학생` 메시지로 추가 → `evaluateAnswer` → 결과를 `티쳐` 메시지로 추가 → TTS 재생.
- 매 메시지 변경 시 `putRecord(today, ...)`로 즉시 영속화 (summary는 항상 `null` 유지).

## 7. 히스토리 + 지연 요약 (`services/summary.ts`, `HistoryScreen`)

- 히스토리 화면 **진입 시**:
  1. `getRecordsBefore(today)`로 이전 기록 조회.
  2. `summary === null`인 항목 필터.
  3. 대상이 있으면 **로딩 화면 표시** → 각 항목 `summarize` → `putRecord`로 저장.
  4. 완료 후 로딩 해제, 목록 렌더링.
  5. 대상이 0건이면 로딩 화면 즉시 사라짐(렌더링 직행).
- 목록: 날짜 내림차순 + 페이지네이션(test.html 시안). 행 클릭 시 하단에 해당 날짜 대화 표시.

## 8. 화면 (test.html 이식)

- `index.css`에 test.html의 스타일(Jua 폰트, 파스텔, 360×640 컨테이너) 그대로 이식.
- `App.tsx`가 `Screen = 'main' | 'chat' | 'history'` 상태로 전환(클래스 토글 대신 조건부 렌더링).
- 클래스명·색상·이모지·문구는 시안 유지.

## 9. 구현 마일스톤

1. **M1 (1차)**: scaffold → 타입/config → IndexedDB → OpenAI 서비스 → TTS/STT(WebSpeech) → 3화면 + 로딩 + 대화 루프. **브라우저에서 전체 시나리오 동작.**
2. **M2 (2차)**: Whisper WASM 폴백 통합, 인식률 기준 자동 전환.
3. **M3**: 다듬기(에러 처리 UX, 마이크 권한 안내, 빈 키 안내 등).
